# Plan-Dokumente

Dieses Verzeichnis sammelt fokussierte Planungsdokumente für bevorstehende Implementierungsschritte in GUSTAV. Ziel ist eine schlanke, nachvollziehbare Vorbereitung (Contract‑First, TDD) vor jeder Codeänderung.

- identity_access.md – Plan für Benutzerverwaltung (Keycloak), inkl. User Stories, BDD, API-Verträge und Migrationsstrategie.
 - auth_forgot_password.md – Plan für den Passwort‑Reset‑Redirect (GET /auth/forgot): User Story, BDD, Tests, Implementierung.
 - auth_register.md – Plan für die Registrierung (GET /auth/register): User Story, BDD, Tests, Implementierung.

Hinweis: Ausführliche Hintergrundinfos zum aktuellen Auth‑Stand finden sich in `../auth_legacy.md`.
